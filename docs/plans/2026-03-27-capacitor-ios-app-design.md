# ProjectIT iOS App — Capacitor Integration Design

**Date:** 2026-03-27
**Status:** Approved

## Goal

Wrap ProjectIT's existing React/Vite web app in a native iOS shell using Capacitor. Publish to the App Store with native features: push notifications, camera, haptic feedback, badge count, and offline mode.

## Architecture

```
ProjectIT (Vite/React) -> npm run build -> dist/
                                             |
                              Capacitor syncs dist/ -> ios/App/
                                             |
                              Xcode builds native iOS app
                                             |
                              Native shell + WKWebView + Capacitor plugins
```

The web app runs inside a native WKWebView. Capacitor bridges JS calls to native iOS APIs. The Railway API backend stays unchanged — `VITE_API_URL` points to `https://api-production-2dbd.up.railway.app`.

## Existing PWA Foundation

Already in place (no changes needed):
- `manifest.json` with app name, icons (72-512px), theme color
- `sw.js` service worker with cache-first for static assets
- Apple meta tags: `apple-mobile-web-app-capable`, `viewport-fit=cover`
- Icons at all required sizes in `public/icons/`

## Native Features

### Phase 1 — Foundation + Core Native

| Feature | Plugin | Purpose |
|---------|--------|---------|
| Push notifications | `@capacitor/push-notifications` | Task assignments, overdue alerts, comments |
| Camera | `@capacitor/camera` | Upload project photos/documents |
| Haptic feedback | `@capacitor/haptics` | Feedback on task complete, swipe, button taps |
| Badge count | `@capacitor/push-notifications` | Unread notification count on app icon |

### Phase 2 — Offline Mode (future)

| Feature | Plugin | Purpose |
|---------|--------|---------|
| Network detection | `@capacitor/network` | Detect online/offline state |
| Local cache | IndexedDB via service worker | Cache tasks/projects for offline access |
| Background sync | Custom sync queue | Queue mutations, sync when back online |

## Key Technical Decisions

### Push Notifications
- Register device token on app launch via `PushNotifications.register()`
- Store device token in Supabase (new `device_tokens` table)
- Railway API sends pushes via Firebase Cloud Messaging (FCM) which bridges to APNs
- Badge count set from push payload + on app resume from unread count API

### Camera Integration
- Replace web file input with native camera/gallery picker on iOS
- `Camera.getPhoto()` returns base64 or URI, upload to existing file upload endpoint
- Falls back to web file picker on non-native platforms

### Haptic Feedback
- `useHaptics()` hook wrapping `Haptics.impact()` / `Haptics.notification()`
- Light impact: button taps, navigation
- Medium impact: task completion, status changes
- Success notification: swipe-to-dismiss

### Platform Detection
```js
import { Capacitor } from '@capacitor/core';

// All native feature usage gated behind this check
if (Capacitor.isNativePlatform()) {
  // native path
} else {
  // web path (existing behavior, unchanged)
}
```

## Web App Changes

Minimal and additive:
- `src/lib/capacitor.js` — platform detection + native utility functions
- `src/hooks/useHaptics.js` — haptic feedback hook
- `src/hooks/usePushNotifications.js` — push registration + listener hook
- `src/hooks/useNativeCamera.js` — camera wrapper hook
- Existing file upload components get native camera option on iOS
- Existing notification system gets push token registration

## Build & Deploy Pipeline

```
Development:   npm run dev              (web, unchanged)
Web deploy:    npm run build && railway up  (unchanged)
iOS build:     npm run build && npx cap sync && npx cap open ios
TestFlight:    Archive in Xcode -> Upload to App Store Connect
```
